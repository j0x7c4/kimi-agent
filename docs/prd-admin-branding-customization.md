## 一、背景

Kimi Code Web UI 是一个基于 Web 的 Agent IDE，面向企业和团队用户提供浏览器自动化和代码执行能力。当前系统中的品牌元素全部硬编码在前端代码中：

- **Logo**: 固定为 `/logo.png`，同时用作浏览器 Favicon
- **品牌名称**: 固定为 "Kimi Code"，出现在侧栏（收起/展开状态）、登录页等位置
- **版本号**: 由构建时常量 `kimiCliVersion` 决定
- **页面标题**: 在 `index.html` 中硬编码为 "Kimi Code Web UI"
- **Logo 点击链接**: 固定指向 `https://www.kimi.com/code`

上述元素由 `kimi-cli-brand.tsx` 组件统一渲染，被 `login-page.tsx`（登录页）和 `sessions.tsx`（主界面侧栏）引用。

对于私有化部署或 OEM 场景，管理员需要将界面品牌替换为自己的标识，而不应修改源码或重新构建镜像。因此需要提供一个管理后台入口，让管理员通过 UI 动态配置品牌信息。

## 二、目标

1. 在管理后台 (`/admin`) 新增 **"Branding"** 标签页，允许管理员在线配置所有品牌元素。
2. 配置保存后 **立即对所有用户生效**，无需重新部署或重启服务。
3. 未做任何定制时，系统行为与当前完全一致（默认值兜底）。
4. 品牌配置数据 **持久化到数据库**，容器重启后配置不丢失。

## 四、功能需求

### 4.1 品牌配置项

| 配置项           | 字段名 (key)      | 类型   | 约束                                             | 默认值                           |
| ---------------- | ------------------ | ------ | ------------------------------------------------ | -------------------------------- |
| 品牌 Logo        | `logo`             | 图片   | PNG / SVG / JPEG，最大 512 KB                    | 内置 `/logo.png`                 |
| 品牌名称         | `brand_name`       | 字符串 | 最多 30 个字符                                   | `"Kimi Code"`                    |
| 版本号           | `version`          | 字符串 | 最多 20 个字符，可为空                           | 空 (回退到内置 `kimiCliVersion`) |
| 页面标题         | `page_title`       | 字符串 | 最多 60 个字符                                   | `"Kimi Code Web UI"`            |
| Logo 点击链接    | `logo_url`         | URL    | 合法 URL，以 `http://` 或 `https://` 开头        | `"https://www.kimi.com/code"`    |
| Favicon          | `favicon`          | 图片   | ICO / PNG / SVG，最大 256 KB                     | 内置 `/logo.png`                 |

### 4.2 管理后台 UI

#### 4.2.1 新增 "Branding" 标签页

在现有 `/admin` 页面的标签栏（目前有 "Users" 和 "Plugins"）中新增第三个标签 **"Branding"**，图标使用 `Palette`（来自 lucide-react）。

#### 4.2.2 表单布局

Branding 标签页内容为一个垂直排列的表单卡片，分为以下区域：

**区域 A — Logo 与 Favicon**

| 元素                 | 说明                                                                                                                                 |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| Logo 上传区域        | 左侧为当前 Logo 预览（64x64），右侧为上传按钮和"恢复默认"按钮。支持点击上传或拖拽上传。上传后立即在预览区显示。                       |
| Favicon 上传区域     | 同上，预览尺寸 32x32。                                                                                                               |
| 格式/大小提示        | 在每个上传区域下方以灰色小字显示允许的格式和最大文件大小。                                                                           |

**区域 B — 文本配置**

| 元素              | 说明                                                                 |
| ----------------- | -------------------------------------------------------------------- |
| Brand Name 输入框 | 单行文本框，placeholder 为 "Kimi Code"，右侧显示字符计数 (x/30)。   |
| Version 输入框    | 单行文本框，placeholder 为内置版本号，右侧显示字符计数 (x/20)。     |
| Page Title 输入框 | 单行文本框，placeholder 为 "Kimi Code Web UI"，右侧字符计数 (x/60)。|
| Logo URL 输入框   | 单行文本框，type=url，placeholder 为默认 URL。                       |

**区域 C — 操作栏**

| 元素         | 说明                                                                                                          |
| ------------ | ------------------------------------------------------------------------------------------------------------- |
| 实时预览     | 表单右侧或上方展示一个迷你预览卡片，模拟侧栏品牌区域的渲染效果（Logo + 品牌名 + 版本号），随表单输入实时更新。|
| 保存按钮     | 主操作按钮 "Save Changes"，仅在表单有修改时高亮可点击。保存成功后显示 toast 提示。                             |
| 重置按钮     | "Reset to Defaults" 按钮，弹出二次确认对话框后清除所有自定义配置，恢复默认值。                                 |

### 4.3 权限控制

- **仅管理员** (role = "admin") 可访问 Branding 标签页及对应的写入 API。
- 读取品牌配置的公开 API 无需身份验证（用于登录页等未登录场景）。

### 4.4 配置生效机制

| 场景                     | 生效方式                                                                                                   |
| ------------------------ | ---------------------------------------------------------------------------------------------------------- |
| Logo / 品牌名 / 版本号  | 前端在应用启动时调用公开 API 获取品牌配置并缓存到 React Context；管理员保存后通过 Context 更新，立即生效。  |
| 页面标题 (document.title)| 前端通过 `useEffect` 监听品牌配置变化并动态设置 `document.title`。保存后立即生效。                          |
| Favicon                  | 前端动态修改 `<link rel="icon">` 的 `href`。部分浏览器需要刷新才能看到变化，可接受。                       |

### 4.5 默认行为

当数据库中没有任何品牌配置记录时（包括全新部署），系统行为与当前硬编码版本完全一致：

- Logo 显示 `/logo.png`
- 品牌名称显示 "Kimi Code"
- 版本号显示内置 `kimiCliVersion`
- 页面标题为 "Kimi Code Web UI"
- Logo 链接指向 `https://www.kimi.com/code`
- Favicon 使用 `/logo.png`

### 4.6 数据持久化

- 品牌配置存储在现有 SQLite 数据库（`users.db`）中，新增 `branding` 表。
- 图片（Logo、Favicon）以 **Base64 编码字符串** 存储在数据库字段中，简化部署（无需额外文件存储或挂载卷）。
- 数据库路径由 `KIMI_SHARE_DIR` 环境变量控制，已挂载持久卷，容器重启后数据不丢失。

#### 数据库表结构

```sql
CREATE TABLE IF NOT EXISTS branding (
    key   TEXT PRIMARY KEY,
    value TEXT NOT NULL
);
```

每个配置项对应一行记录，`key` 为配置字段名（如 `logo`、`brand_name` 等），`value` 为字符串值（图片为 `data:image/png;base64,...` 格式的 Data URL）。

## 五、非功能需求

| 类别       | 要求                                                                                                                         |
| ---------- | ---------------------------------------------------------------------------------------------------------------------------- |
| 性能       | 公开品牌 API 响应时间 < 100ms；Logo Base64 数据不超过 512 KB，不会显著增加页面加载时间。                                     |
| 安全       | 写入接口受 `require_admin` 中间件保护；图片上传需在后端校验 MIME 类型和文件大小，防止恶意内容；Logo URL 做 XSS 防护（仅允许 http/https 协议）。|
| 兼容性     | 支持 Chrome、Firefox、Safari、Edge 最新两个大版本。Favicon 动态替换在 Safari 上可能需刷新，可接受。                           |
| 可用性     | 所有输入字段带有明确的 label、placeholder 和校验提示；图片上传失败时显示具体错误原因（格式不对、文件过大等）。                |
| 可维护性   | 品牌配置通过 React Context 统一分发，消费方组件不直接调用 API，便于后续扩展更多配置项（如主题色等）。                         |

## 六、API 概览

### 6.1 公开接口（无需认证）

#### `GET /api/branding`

获取当前生效的品牌配置。登录页、主页面启动时调用。

**响应示例:**

```json
{
  "logo": "data:image/png;base64,iVBORw0KGgo...",
  "brand_name": "Acme IDE",
  "version": "2.1.0",
  "page_title": "Acme IDE - Web",
  "logo_url": "https://www.acme.com",
  "favicon": "data:image/png;base64,iVBORw0KGgo..."
}
```

未设置的字段返回 `null`，前端使用默认值兜底。

**响应码:**

- `200 OK` — 成功返回配置（即使所有字段均为 `null`）

### 6.2 管理接口（需 Admin 认证）

#### `GET /api/admin/branding`

获取当前品牌配置（与公开接口响应格式一致），用于管理后台表单回显。

**响应码:**

- `200 OK`
- `401 Unauthorized` — 未登录
- `403 Forbidden` — 非管理员

#### `PUT /api/admin/branding`

更新品牌配置。请求体为 JSON，仅包含需要更新的字段；传 `null` 的字段表示清除自定义值（恢复默认）。

**请求示例:**

```json
{
  "brand_name": "Acme IDE",
  "page_title": "Acme IDE - Web",
  "logo": "data:image/png;base64,iVBORw0KGgo...",
  "favicon": null,
  "version": "",
  "logo_url": "https://www.acme.com"
}
```

**校验规则:**

| 字段         | 校验                                                      |
| ------------ | --------------------------------------------------------- |
| `logo`       | 必须为合法 Data URL（`data:image/(png|svg+xml|jpeg);base64,...`），解码后 <= 512 KB；或 `null` |
| `brand_name` | 字符串长度 <= 30；或 `null`                               |
| `version`    | 字符串长度 <= 20；或 `null` / 空字符串                    |
| `page_title` | 字符串长度 <= 60；或 `null`                               |
| `logo_url`   | 以 `http://` 或 `https://` 开头的合法 URL；或 `null`      |
| `favicon`    | 必须为合法 Data URL（`data:image/(x-icon|png|svg+xml);base64,...`），解码后 <= 256 KB；或 `null` |

**响应码:**

- `200 OK` — 更新成功，返回完整的最新配置
- `400 Bad Request` — 校验失败，返回具体错误字段和原因
- `401 Unauthorized`
- `403 Forbidden`

### 7.2 品牌组件 (kimi-cli-brand.tsx) 改造

现有 `KimiCliBrand` 组件改为从 React Context（`BrandingContext`）读取配置：

- `logo`: 优先使用 Context 中的自定义 Logo Data URL，无则使用 `/logo.png`
- `brand_name`: 优先使用自定义名称，无则使用 "Kimi Code"
- `version`: 优先使用自定义版本，无则使用 `kimiCliVersion`
- `logo_url`: 优先使用自定义 URL，无则使用默认 URL

### 7.3 登录页

登录页在渲染前先调用 `GET /api/branding` 获取品牌数据（此接口无需认证），并据此渲染 Logo、品牌名称和页面标题。
